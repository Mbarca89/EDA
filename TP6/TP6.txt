	i.Para ordenar un ArrayList en Java, debes usar clases contenedoras 
	(wrapper) como Integer o Double, ya que los tipos primitivos (int, double)
	no se pueden almacenar directamente en una colección.

	ii.Java utiliza el algoritmo TimSort para ordenar un ArrayList de objetos 
	cuando se usa Collections.sort() o ArrayList.sort().

	iii.En Java, los métodos principales para buscar elementos en un ArrayList
	son contains() (para saber si el elemento existe) e indexOf() (para 
	conocer su posición).

	iv.Si el HashMap de Java encuentra colisiones en un mismo bucket, utiliza
	una combinación de encadenamiento separado (recorrido secuencial mediante
	nodos) y árboles balanceados (desde el JDK 8 en adelante para casos de 
	alta colisión).

	v.No, el método .contains() en Java nunca utiliza búsqueda binaria. Por 
	defecto, está diseñado para ser de uso general en colecciones que pueden 
	estar desordenadas, por lo que siempre realiza una búsqueda lineal con 
	una complejidad de (O(n). Internamente, llama al método .indexOf() y 
	recorre elemento por elemento comparándolos con el objeto buscado.

	vi.Los métodos tradicionales de ordenamiento (como Quicksort o Mergesort)
	no son aplicables en redes porque requieren tener todos los elementos 
	disponibles de antemano y comparar sus valores (a < b) para organizarlos.
	En internet, el flujo es continuo, infinito y los paquetes viajan por rutas
	distintas. El protocolo TCP utiliza una estrategia orientada a la 
	reconstrucción basada en números de secuencia y almacenamiento en 
	búfer:

	-  Numeración secuencial: Cada segmento de datos lleva un encabezado 
	con un número de secuencia.

	-  Recepción y almacenamiento en búfer: Al llegar fuera de orden, TCP no 
	los descarta ni los procesa de inmediato. Los almacena temporalmente en 
	la memoria o búfer del receptor y espera los paquetes faltantes.

	-  Reensamblaje y confirmación: Una vez completada la secuencia, TCP 
	reordena los datos y entrega el flujo continuo a la capa de aplicación.

	-  Control de pérdidas: Si un número de secuencia se salta (por ejemplo, 	
	llega el 1, el 2 y luego el 4), TCP envía confirmaciones duplicadas (ACKs) 
	para solicitar al emisor el reenvío del paquete 3.

c] El promt que generamos  para pedirle a la IA fue: "Necesito un algoritmo en Java que ordene un arreglo de números enteros utilizando QuickSort. 
Quiero que me muestres el código completo, comentado y explicado paso a paso para poder entender 
cómo funciona cada parte del programa."

e]El algoritmo toma el primer elemento del arreglo como pivote. Luego compara los demás valores para colocarlos a la izquierda si son menores o a la derecha si son mayores.
 Cuando el pivote queda ubicado en su posición correcta, el algoritmo repite el proceso con las dos partes generadas. 
Esto continúa hasta que todos los elementos quedan ordenados.

f] " El QuickSort es más rápido que Bubble Sort e Insertion Sort cuando hay muchos datos para ordenar, porque va separando el arreglo en partes más chicas y las ordena por separado.
 Igual, si el pivote que elige no es bueno, puede tardar más y perder rendimiento."

g]QuickSort responde a la estrategia de Divide y Vencerás, porque lo que hace es agarrar el arreglo, dividirlo en partes más chicas usando un pivote y después ordenar cada parte por separado.
 Va repitiendo ese proceso hasta que todos los números quedan ordenados.
